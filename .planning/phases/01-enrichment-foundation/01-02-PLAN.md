<feature>
  <name>Context Extraction</name>
  <files>tools/enrich_entities.py, tests/test_enrich_entities.py</files>
  <behavior>
    extract_entity_context(srt_cues: list, entity_occurrences: list, window_cues: int = 3) -> str

    For each occurrence:
    1. Find the cue by cue_idx (1-based index matching SRT)
    2. Extract window_cues cues before and after (default 3 = ~100-150 words)
    3. Join cue texts with spaces
    4. Strip any "Speaker N" lines (already handled by srt_entities.py, but double-check)
    5. Collapse whitespace

    For multiple occurrences:
    1. Extract context for each occurrence
    2. Detect overlapping windows (if occurrence cue indices are within 2*window_cues of each other)
    3. Merge overlapping windows to avoid duplicate text
    4. Join non-overlapping contexts with " [...] " separator

    Test cases:
    - Single mention: returns surrounding cues text
    - Two mentions far apart: returns both contexts joined with " [...] "
    - Two mentions close together (overlapping): merged into single context
    - Edge case: first cue (no preceding cues): returns available context
    - Edge case: last cue (no following cues): returns available context
    - Edge case: cue_idx not found: skips that occurrence
  </behavior>
  <implementation>
    1. Add tests to tests/test_enrich_entities.py for context extraction
    2. Create mock SrtCue dataclass or use namedtuple for testing (match srt_entities.py structure)
    3. Add to tools/enrich_entities.py:
       - extract_single_context(srt_cues: list, cue_idx: int, window_cues: int) -> tuple[int, int, str]
         Returns (start_idx, end_idx, context_text) for overlap detection
       - merge_context_windows(contexts: list[tuple[int, int, str]]) -> str
         Merges overlapping windows, joins non-overlapping with " [...] "
       - extract_entity_context(srt_cues: list, entity_occurrences: list, window_cues: int = 3) -> str
         Main function combining above
    4. Use existing parse_srt from srt_entities.py - DO NOT re-implement SRT parsing
    5. Window of 3 cues before + 3 cues after = 7 cues total = ~100-150 words (typical cue is 15-25 words)
  </implementation>
</feature>
