<task type="auto">
  <name>Task 1: Add strategy iteration logic to download_entity</name>
  <files>tools/download_entities.py</files>
  <action>
Modify download_entity() function to use search strategies:

1. Add get_search_terms(entity_name, payload) helper function:
   - Check if payload has search_strategies field
   - If yes: return list of terms to try:
     - First: best_title (if exists and validated)
     - Then: each query from validated_queries (in order)
     - Finally: original entity_name as fallback
   - If no search_strategies: return [entity_name] (backward compatible)

2. Update download_entity() function signature:
   - Add optional parameter: search_strategies_enabled=True

3. Modify download_entity() logic:
   - Get search_terms from get_search_terms()
   - Try EACH search term (not stop on first success):
     - Call existing wikipedia_image_downloader.py with search term
     - Record result: {term, success, entity_dir, images_count}
   - After all attempts, pick best result:
     - Prefer result matching best_title (exact match)
     - Otherwise prefer first successful result
     - If none succeeded, mark as failed

4. Return tuple now includes matched_term:
   - (entity_name, success, entity_dir, matched_term)
   - matched_term is the search term that succeeded (or None if failed)

5. Track which strategy was used:
   - "best_title" if matched best_title
   - "query_N" if matched Nth query
   - "fallback" if matched original entity name
   - None if no match

Note: The existing downloader uses entity_dir = out_dir / safe_folder_name(entity_name). When trying different search terms, we should still use entity_name for the output folder (not the search term) to maintain consistent folder naming.
  </action>
