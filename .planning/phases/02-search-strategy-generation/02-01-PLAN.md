<task type="auto">
  <name>Task 2: Implement LLM strategy generation with batch processing</name>
  <files>tools/generate_search_strategies.py</files>
  <action>
Add to generate_search_strategies.py:

1. generate_batch_strategies(entities, video_context, client) function:
   - Takes list of entity dicts (from enriched_entities.json) with name, entity_type, context
   - Takes video_context string (video topic/title)
   - Takes Anthropic client instance
   - Constructs prompt including:
     - Video topic for disambiguation context
     - Entity list with name, type, and transcript context
     - Instructions: People get 3 queries, others get 2; order by confidence; include disambiguation hints only when ambiguous
   - Uses client.beta.messages.parse() with betas=["structured-outputs-2025-11-13"]
   - Model: claude-sonnet-4-5-20250929
   - output_format=BatchSearchStrategies
   - Returns list of SearchStrategy objects

2. generate_search_strategies(enriched_map, video_context, batch_size=7) main function:
   - Loads ANTHROPIC_API_KEY from environment
   - Processes entities in batches of batch_size (default 7, configurable 5-10)
   - For each batch:
     - Calls generate_batch_strategies
     - On success: merge strategies into entities
     - On batch failure: retry individually with tenacity (3 attempts, exponential backoff)
     - On individual failure: mark entity with status="failed_generation", use entity name as fallback
   - Returns enriched_map with search_strategies field per entity

3. Each entity gets search_strategies dict:
   ```json
   {
     "best_title": "Albert Einstein",
     "queries": ["Albert Einstein", "Einstein physicist", "Albert Einstein scientist"],
     "confidence": 9,
     "status": "generated"  // or "failed_generation"
   }
   ```

Use tenacity for retry logic: @retry(stop=stop_after_attempt(3), wait=wait_exponential(min=1, max=10))
  </action>
